# Q2a. A file has permissions rw-r----- and is owned by user alice and
# group mlteam. Can user bob, who is a member of mlteam, execute the
# file? Can he read it?

Q2a: Bob cannot execute the file, but he can read it.


# Q2b. You run ls -l and see -rwxrwxrwx on a file containing your API
# keys. Explain why this is a problem and what permissions you would
# set instead. Give the chmod command you would use.

Q2b: This is bad because it means all users can read, write, and execute
     the file containing your API keys.  You don't want just anyone to
     be able to use these, and you certainly don't want anyone to be able to 
     change or delete the file.  It makes most sense for only you to be 
     able to read them and change them.
	chmod 600 <filename>
     This would give rw-------


# Q2c. A directory has permissions rwxr-xr-x. You are not the owner
# and not in the group. Can you list the directory's contents? Can you
# create a new file inside it? Explain why or why not.

Q2c:    You can list the directories contents because you have read 
	permissions as an "other" user.  You cannot create a file inside 
	of it because you do not have write permissions.
 

# Q2d. What is the difference between chmod 755 and chmod u=rwx,go=rx?

Q2d:  There is no difference, they both provide all permissions to
      the file owner, and read and execute permissions to the file's
      group and all other users.
